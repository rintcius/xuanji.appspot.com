Introduction: how to write down functions

We'll begin our introduction to the lambda calculus by asking how we might write down a function. There are standard ways to notate numbers,

42
0x2a

sets,

{0,1,2}
{x | x > 0, x ∈ R}

etc, but there doesn't seem to be a way to write down functions.

"But there is a way!" you say. "for example to write down the function f that triples its argument - f(1) = 3, f(4) = 12, etc - we can just write:

f(x) = 3x

."

Well, this approach suffers the drawback that you must give the function a name. What if your function returns another function? And you're not really writing down the function as much as writing down an equation that it satisfies. We need a notation for anonymous functions - functions that don't have names.

In lambda calculus, we would write f as

λx.3x

Where, in general, a function that takes an argument and returns something is written as

λ[argument].[return value]

Simple examples

Now we have a universe of functions that we can talk about. The next logical step would be to introduce something for the functions to act on - perhaps numbers. Lists might be useful, for building data structures.

I'm not going to do that. Our universe of discourse will only contain functions. What do they act on? Other functions. Let's see how far we can take this idea. It might seem at first that there aren't any interesting functions to write down. How about one that does nothing?

The identity function is a function I that always returns its argument

I = λt.t
((lambda t t) y)

The constant function, denoted by Cr , is a function that always returns r , is defined by

Cr = λt.r
((lambda t r) x)

What does the function

K = λr.(λt.r)

do? It returns the constant function of its argument - Kr = Cr.
